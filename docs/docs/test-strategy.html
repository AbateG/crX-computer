Test Strategy

Orchestrix uses a layered testing approach:

  • Unit Tests: Validate individual components and business logic
  • Integration Tests: Ensure correct interaction between modules and with the database
  • E2E Tests: Simulate real user scenarios and workflows
dotnet test
# For E2E (Cypress):
cd src/Tests/E2E
npx cypress open
