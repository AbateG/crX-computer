Architecture

Orchestrix uses Clean Architecture principles for maintainability and scalability. The solution is organized into:

  • Domain: Core business logic and entities
  • Application: Use cases, commands, queries, and validation
  • Infrastructure: Data access, external services, and integrations
  • API: RESTful endpoints, controllers, and presentation
